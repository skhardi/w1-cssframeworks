Week 1: CSS Frameworks

6/16/2021

About

We’ll be starting off relatively easy as we ease into things - since you’ve all demonstrated a pretty thorough understanding of basic HTML/CSS, you’ll likely already be familiar with this concept: CSS frameworks. They are essentially libraries of components which are used as “building blocks” for a website - they utilize pre-defined CSS stylesheets which usually include JavaScript for functionality. Most will even have their own layout/grid system, which can largely change how your HTML is written and structured.

Today, we’re going to be working with the very popular Bootstrap framework. It was created by Twitter to be a mobile-first framework - or, a responsive framework. As such, Bootstrap-created components are generally created to appear best on mobile - and then scale to desktop application as well. This is significant in today’s world as since 2017, mobile traffic has made up the majority of internet traffic - especially on e-commerce sites in which web developers are most in demand to manage.

Activity

Today will be very straightforward, simple, and open-ended to start off with our first activity. We won't have a pre-recorded screen share this week, but moving forward into more complex technologies, you can expect to see pre-recorded screen shares or live screen shares/meetings to accompany weekly tasks. This time, though, we'll mainly just be reading some documentation :)

First, create a Git repository in this organization. Make sure to create it with a README.md so the repository isn't empty. Steps to clone and initiate in VSCode (it will likely prompt you to sign-in at one point if not signed into Bash already):

  • Green Code button on YOUR GitHub repo's page
  • HTTPS should be selected by default - copy the link
  • In Git Bash (or in VSCode's integrated terminal - make sure you're on bash and not PowerShell), use ls to view the directory you are in, and cd <directory name> to navigate to the directory you want your repo to live in. You can use cd .. to move UP a directory.
  • Use the command git clone in your directory of choice
  • Navigate again to the freshly cloned repo with cd
  • Once your Git terminal is in the repo folder, you can use code . to open it in VSCode.

There are plenty of ways to do this - the main goal is having your repository open in your IDE so you can do commits, pushes, and pulls with just the IDE's integrated terminal. As you likely know, this is absolutely key to good software development practice and will make version control a LOT easier.

Using the template index.html file in this repository, your task this short week is to simply pick any website of your choice - using Bootstrap's grid system and components, replicate as best as possible any page or pages you’d like to the best of your ability - don’t worry too much about styling the page elements to fit the original page. Focus more on the page layout and components rather than the actual content, sizing, or styling.

Be especially mindful of how Bootstrap’s grid system works - it is likely that future work with React component libraries will employ similar grid systems. Click here to view their grid system documentation. Pay attention to how these <div> tags for rows and columns especially are structured - this is a quite common grid structure syntax in many React libraries, including the ubiquitous Material-UI, the most popular React library of Google's famous Material Design.

Feel free to explore the documentation and all of the different component variations and utilities section. There's a lot of customizability with stock Bootstrap components. Finally, once you are comfortable with Bootstrap's grid system and component implementation, make yourself a little "about me" single-page landing page - tell us all about yourself, any of your passions, or even make yourself a portfolio! The format, content, everything is completely up to you. Get as creative or as practical as you'd like. If you still find yourself done early, feel free to poke around the internet and take a look at some other HTML/CSS frameworks out there and how they work. Completely up to you, but if you'd like, you can try creating your "about" page using another CSS framework instead of Bootstrap. You'd be surprised how similar implementation with these simple frameworks is with more dynamic front-end frameworks like Angular and React's JSX.

Here is a link to Bootstrap's component section in their documentation - you'll find most components in the left-hand column. Remember, as programmers, documentation is everything! Make sure to view the section below for documentation, and how to install Bootstrap's framework.

Once you're done, simply commit and push your files to the Git repository you made and drop a link in the Google Chat room!
